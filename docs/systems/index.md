# Systems

This section documents the places where work happens: servers, fileshares, XNAT, CCAD systems, Jubail, containers, and other infrastructure.

Use this section when the question is "where is this?", "how do I access it?", or "which system owns this part of the workflow?"

## Main Systems

| System | What belongs here |
|--------|-------------------|
| XNAT | XNAT projects, pipeline nodes, storage, and automation |
| CCAD | VPN, VDI, PACS, BrainLab, Asif's iMac, and CCAD-specific data movement |
| HPC / Jubail | Cluster login, SLURM, modules, scratch, and running jobs |
| Containers | Docker, Singularity / Apptainer, and running containerized pipelines |
| Fileshares | MRI projects, CTP-XNAT, fMRI fileshare, Box archives, and local mount points |

## Practical Rule

If a page is mostly about access, addresses, mounted paths, servers, or system ownership, it belongs here. If a page is mostly about doing a task step by step, it belongs in [Workflows](../workflows/index.md).
